# 沉浸式论文翻译辅助阅读器 – 需求分析报告

## 1. 项目概述

### 1.1 项目背景
随着全球科研交流日益频繁，英文论文成为获取前沿知识的主要载体。然而，对于大量非英语母语的研究人员、研究生及高年级本科生而言，语言障碍依然是高效阅读文献的主要瓶颈。  
因此，开发一款**沉浸式论文翻译辅助阅读器**，将翻译、结构梳理、阅读管理融为一体，具有明确的实用价值与推广前景。

### 1.2 项目目标
打造一个集论文导入与管理、智能大纲生成、逐句翻译、重点内容总结于一体的阅读工具，帮助用户克服语言障碍，提升文献阅读效率与深度。

---

## 2. 用户需求分析

### 2.1 目标用户
- 非英语母语的研究人员、博士生、硕士生、高年级本科生
- 需要频繁阅读英文论文的科研团队或实验室
- 希望提升文献管理效率的个人或机构

### 2.2 核心用户需求
| 需求类型 | 描述 |
|---------|------|
| 语言辅助 | 准确、低干扰的逐句翻译，支持术语自定义 |
| 结构导航 | 自动提取论文大纲，帮助快速理解全文框架 |
| 知识沉淀 | 自动总结已读部分的核心观点，支持笔记导出 |
| 文献管理 | 个人文献库、分类、标签、搜索、进度同步 |

---

## 3. 功能需求（当前版本）

### 3.1 论文导入与存储

| 子功能 | 描述 |
|--------|------|
| 支持格式 | 优先支持 PDF，后续扩展 EPUB、网页链接 |
| 导入方式 | 本地文件上传、拖拽导入、批量导入 |
| 存储管理 | 个人文献库，支持分类、标签、搜索；云端存储（可选），多端同步 |
| 阅读进度 | 记录每篇论文的已读比例、最后阅读位置 |

### 3.2 自动总结论文大纲

| 子功能 | 描述 |
|--------|------|
| 技术手段 | 基于大语言模型（LLM）对论文全文进行结构化解析 |
| 输出内容 | 提取章节结构（如 Introduction、Method、Results、Conclusion）；每个章节生成 1–2 句摘要 |
| 展示方式 | 树形或层级视图，支持点击跳转对应段落 |
| 用户价值 | 帮助读者在深入阅读前快速建立论文整体认知 |

### 3.3 逐句翻译

| 子功能 | 描述 |
|--------|------|
| 交互方式 | 单击句子查看详细翻译，或 AB 栏对比全篇翻译 |
| 翻译能力 | 调用通用翻译 API（DeepL、GPT）或本地部署模型；保留原文排版与图表位置；支持术语库自定义 |
| 沉浸式设计 | 低干扰界面，翻译作为辅助层；可切换“仅原文”“仅译文”“对照”三种模式 |

### 3.4 总结看过论文的重点

| 子功能 | 描述 |
|--------|------|
| 自动提取 | 每篇论文通过 LLM 自动提取已读部分的核心观点、关键数据、研究方法 |
| 用户补充 | 用户可通过 comment 形式添加个人理解或标注 |
| 展示方式 | 卡片或要点列表形式呈现 |
| 导出功能 | 支持将重点内容导出为 Markdown / PDF 格式 |

---

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 论文打开时间 < 3 秒；翻译响应 < 1 秒/句 |
| 可用性 | 界面简洁，沉浸式设计，支持暗色模式；操作路径不超过 3 步 |
| 兼容性 | 支持 Windows、macOS、Linux Web 端；后续适配平板与移动端 |
| 可扩展性 | 模块化设计，便于后续增加新的 LLM 或翻译引擎 |
| 数据安全 | 本地优先，云端存储可选且需用户授权；用户笔记与文献库加密传输 |
| 隐私保护 | 用户数据不上传第三方（除用户主动使用的翻译 API 外） |

## 5. 未来计划

以下功能计划在后续版本中逐步实现，以进一步提升系统的智能化与个性化体验：

### 5.1 Agent（智能代理）模块

Agent 是一个基于 LLM 的对话式智能助手，能够感知用户阅读行为，主动提供上下文相关的帮助，并响应用户的自然语言指令。

| 功能 | 描述 |
|------|------|
| **阅读问答助手** | 用户可随时向 Agent 提问（如：“这个方法为什么要用 ANOVA？”），Agent 基于论文内容及用户已读部分进行回答 |
| **术语解释与推荐** | 当用户选中专业术语时，Agent 自动提供定义、背景知识或相关论文链接 |
| **阅读提醒与计划** | Agent 根据用户阅读进度和习惯，主动提醒或建议下一步阅读顺序 |
| **跨论文关联推荐** | Agent 分析文献库中的论文，发现主题、方法或引用关系，主动推荐相关章节 |
| **个性化翻译优化** | Agent 学习用户所在领域的术语偏好，动态调整翻译结果 |
| **笔记辅助生成** | 用户读完论文后，Agent 协助生成结构化笔记草稿（研究问题、方法、结果、局限） |
| **语音交互** | 支持语音输入与 Agent 对话，提升移动端或沉浸阅读时的便捷性 |

### 5.2 其他扩展计划
- 支持更多文献格式（EPUB、网页链接）
- 平板与移动端适配
- 团队协作与共享文献库
- 与主流文献管理工具（Zotero、Mendeley）双向同步

---

## 6. 总结

本需求分析报告定义了“沉浸式论文翻译辅助阅读器”当前版本的核心功能：论文导入与管理、自动大纲生成、逐句翻译、重点内容总结。同时，将 **Agent 智能代理模块** 列入未来计划，作为产品的差异化竞争力与长期演进方向。下一步可基于本报告进行系统设计与原型开发。
