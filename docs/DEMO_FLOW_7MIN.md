# Impulse 7 分钟演示流程

---

## 演示前准备

### 环境检查

- [ ] Chrome 浏览器已安装 Impulse 扩展（开发模式加载 `build/chrome-mv3-dev`）
- [ ] 扩展图标固定在工具栏
- [ ] 已配置 API Key（DeepSeek 或 OpenAI）
- [ ] 已准备好 2 篇 PDF 论文（建议 ArXiv 论文，一篇已读过、一篇未读）
- [ ] 网络连接正常
- [ ] 侧边栏宽度已调整为合适大小

### 推荐演示论文

提前在浏览器中打开 1-2 篇，预热阅读历史：

| 用途 | 建议 |
|------|------|
| 主打演示 | 一篇经典的 AI/ML 论文（如 "Attention Is All You Need"）|
| 对比演示 | 一篇相关方向的论文（如 BERT 或 ViT）|

### 演示前 5 分钟操作

1. 打开一篇 PDF，用 Impulse 查看器阅读，随意选中几段文字
2. 用高亮功能标记 2-3 处（展示阅读历史时会有数据）
3. 保存 1 条笔记

## 详细演示脚本

### 0:00-0:30 | 开场（30 秒）

**说**：

> "大家好，我是 Impulse 的开发者。大家都读过论文吧？打开一篇 20 页的英文论文，密密麻麻的文字，不确定是否相关，还有五篇等着你。今天我要展示的 Impulse，就是解决这个问题的——它不是一个简单的工具，而是一个住在浏览器侧边栏里的 AI 研究助手。"

**屏幕**：展示 Impulse Logo / 扩展图标

**要点**：
- 建立共鸣（所有人都读过论文）
- 一句话定位：AI 研究助手，在侧边栏里

---

### 0:30-1:00 | 启动（30 秒）

**操作**：
1. 在 Chrome 中打开一篇 PDF 论文（用 ArXiv 链接）
2. 展示 Impulse 自动检测 PDF 并提示"在 Impulse 查看器中打开"
3. 点击打开 → 进入内置 PDF 阅读器

**说**：

> "当你打开一篇 PDF，Impulse 会自动接管。它内置了一个 PDF 阅读器，支持缩放、翻页，最关键的是——它能捕获你的选中文字，并实时发送给侧边栏的 AI。"

**屏幕**：
- 展示 PDF 阅读器的 Canvas 渲染
- 缩放操作（Ctrl+滚轮）
- 选中一段文字 → 展示文字高亮

**要点**：
- 内置 PDF 查看器 = 完整控制权
- 选中即通信（无需复制粘贴）

---

### 1:00-2:30 | 助手模式（90 秒）

**操作**：点击工具栏图标，打开侧边栏

#### 1:00-1:20 | 摘要（20s）

**操作**：
1. 在 PDF 中选中一段 Methods 文字
2. 侧边栏自动同步选中文本
3. 选择阅读目标为"了解方法"
4. 点击"摘要"按钮

**说**：

> "侧边栏有 5 个功能标签。首先看摘要——我选中一段 Methods，选择一个阅读目标，AI 会根据我的目标调整摘要的侧重点。选择了'了解方法'，它就会聚焦在技术路线和算法上。"

**屏幕**：展示 AI 生成的中文摘要，重点标注技术方法部分

#### 1:20-1:35 | 翻译（15s）

**操作**：
1. 切换到翻译标签
2. 使用同一段选中文字
3. 点击翻译

**说**：

> "翻译标签提供学术级别的中英互译，保留术语的准确性。"

**屏幕**：展示翻译结果，对比原文

#### 1:35-2:00 | 高亮（25s）

**操作**：
1. 切换到高亮标签
2. 点击"智能高亮"→ AI 提取 3-7 个关键短语
3. 展示 PDF 页面上出现分类颜色标记
4. **右键菜单演示**：在 PDF 中选中一句话 → 右键 → 选择"🔴 重要"→ 红色高亮出现
5. 展示 5 种颜色的分类

**说**：

> "高亮是最直观的功能。AI 自动提取关键短语，然后用不同颜色标记到 PDF 上——红色重要、蓝色疑问、绿色定义、紫色方法、黄色默认。你也可以手动选中文字，右键直接标记。所有高亮都是持久化的，下次打开还在。"

**屏幕**：
- PDF 页面上展示 5 色高亮
- 右键菜单的 5 个选项
- 侧边栏高亮列表

#### 2:00-2:20 | 笔记 + 问答（20s）

**操作**：
1. 切换到笔记标签
2. 选中一段文字 → 添加评论 → 保存
3. 切换到问答标签
4. 输入"这篇论文的主要贡献是什么？"

**说**：

> "笔记和问答让阅读变成双向的。选中文字直接加笔记，问答标签支持多轮对话，AI 会带着论文上下文回答你的问题。"

#### 2:20-2:30 | 快速操作工具栏（10s）

**操作**：
1. 在 PDF 页面中选中一段文字
2. 展示浮动的快捷工具栏（翻译 T / 摘要 S / 术语 E / 高亮 H / 笔记 N）
3. 点击一个按钮

**说**：

> "还有一个细节——选中文字后会弹出一个浮动工具栏，5 个快捷键，不用切换到侧边栏就能快速操作。"

---

### 2:30-4:30 | Agent 模式（120 秒）⭐ 核心亮点

**操作**：点击 Header 中的"Agent"切换到 Agent 模式

#### 2:30-3:00 | 基础对话 + 工具调用（30s）

**操作**：
1. 输入："帮我总结一下这篇论文的核心方法，然后保存为笔记"
2. Agent 开始工作
3. 展示工具调用过程（侧边栏中显示 `🔧 summarize_selection → 完成`、`🔧 save_note → 完成`）

**说**：

> "现在切换到 Agent 模式——这是 Impulse 最特别的地方。它不是那种只会聊天的 AI。它有 14 个工具，能真正帮你做事。我说'总结核心方法并保存为笔记'——你看，它自动调用了摘要工具，然后把结果保存为笔记。一步到位。"

**屏幕**：
- Agent 对话界面
- 工具调用的状态指示器
- 笔记列表中自动出现新笔记

#### 3:00-3:30 | 多步骤任务（30s）

**操作**：
1. 输入："把论文里的所有定义都高亮出来，用绿色标记"
2. Agent 自动：提取文本 → 识别定义 → 逐个高亮

**说**：

> "更厉害的是，它能做多步骤的任务。'把所有定义用绿色高亮出来'——这需要先理解文本，找到定义，然后一个一个标记到页面上。它自己规划，自己执行。"

**屏幕**：PDF 页面上逐个出现绿色高亮

#### 3:30-3:50 | 技能系统（20s）

**操作**：
1. 在 Agent 对话框输入 `/`
2. 展示 9 个技能的搜索结果
3. 选择 `/contribution`
4. Agent 使用贡献提取技能分析论文

**说**：

> "输入斜杠可以调用预设的技能——9 个专门针对论文阅读场景设计的提示词。比如 `/contribution`，它用一套专门的提示词，把论文的核心问题、方法和结果一次性提取出来。"

**屏幕**：技能搜索结果 + 技能输出结果

#### 3:50-4:10 | Agent 主动提问（20s）

**操作**：
1. 输入："帮我比较一下这篇和我最近读的论文"
2. Agent 弹出提问框："你想从哪个维度比较？方法、实验结果、还是创新点？"

**说**：

> "Agent 不是盲目的——当它需要更多信息时，会主动问你。'你想从哪个维度比较？'——这是协作，不是单向的命令。"

**屏幕**：AskUserQuestionModal 弹出

#### 4:10-4:30 | 阅读目标影响 Agent 行为（20s）

**操作**：
1. 切换阅读目标为"准备引用"
2. 输入："帮我看看这篇论文"
3. Agent 输出的摘要自动偏向引用相关内容

**说**：

> "还记得之前的阅读目标吗？在 Agent 模式下它也生效。选择'准备引用'，Agent 就会自动帮你找引用素材和代表性结论。"

---

### 4:30-5:30 | 论文对比（60 秒）

**操作**：
1. 在 Agent 中输入 `/compare`
2. Agent 列出所有候选论文（从阅读历史中）
3. 选择两篇论文进行对比
4. Agent 从 6 个维度生成结构化对比表格

**说**：

> "读过几篇论文后，最头疼的就是对比。用对比技能，Agent 从阅读历史中找出你读过的论文，然后从贡献、方法、实验、局限性、创新性、实用价值六个维度生成结构化对比。"

**屏幕**：
- 候选论文列表
- 6 维对比表格
- 保存对比结果

---

### 5:30-6:00 | 阅读历史（30 秒）

**操作**：
1. 展示 Agent 在对话中引用了阅读历史
2. 输入："我上次读这篇是什么时候？"
3. Agent 返回具体时间和阅读时长

**说**：

> "Impulse 用 SQLite 在浏览器里记录了你的阅读历史——读了哪些论文、读了多久、什么时候读的。所以 Agent 可以说'你上次读这篇是 3 天前，一共花了 25 分钟'。所有这些数据都在本地，不上传。"

**屏幕**：
- Agent 返回阅读统计
- 强调本地存储、隐私安全

---

### 6:00-6:30 | 多 Provider + 可扩展性（30 秒）

**操作**：
1. 打开设置 → 展示 4 种 Provider 预设（OpenAI / DeepSeek / Qwen / Custom）
2. 切换 Provider → 无需改代码
3. 可选：展示 Bug 报告功能的 AI 诊断

**说**：

> "Impulse 支持多种 LLM——OpenAI、DeepSeek、Qwen，或者任何兼容的接口。一键切换。而且它还有智能 Bug 报告系统，遇到问题可以一键收集诊断信息，AI 会帮你分析根因。"

**屏幕**：ConfigModal 的 Provider 切换 + Bug 报告界面

---

### 6:30-7:00 | 收尾（30 秒）

**说**：

> "总结一下——Impulse 把 AI Agent 放进了浏览器侧边栏，让你读论文的时候：不用切换工具、不用复制粘贴、不用手动整理。它帮你高亮、帮你做笔记、帮你对比论文、记住你的阅读历史。
>
> 技术栈是 Plasmo + React + TypeScript，Manifest V3 标准。如果你感兴趣，代码在 GitHub 上开源。
>
> 一句话：**Read less, understand more. 读得更少，理解更多。**谢谢！"

**屏幕**：回到 Impulse Logo + GitHub 地址

## 演示技巧

### 话术要点

| 场景 | 怎么说 |
|------|--------|
| LLM 响应慢 | "AI 正在处理，这正好说明它在做多步骤推理" |
| 高亮没出现 | 切换到一个已有高亮的页面展示，说"有时候 PDF 加载需要刷新" |
| 网络问题 | 提前准备好离线截图作为备用 |

### 引导动作

- 每个功能演示完快速过渡，不要纠结细节
- Agent 模式是最重要的，给它最多的时间
- 让观众看到"发生了什么"（PDF 上的变化）比"说了什么"更重要

### 应急预案

| 风险 | 应对 |
|------|------|
| API 超时 | 提前录制一段 Agent 工具调用的 GIF 备用 |
| PDF 加载失败 | 换一个更小的 PDF（< 5MB）|
| Chrome 崩溃 | 准备一台备用电脑或虚拟机 |
| 网络断开 | 展示已保存的高亮、笔记等离线可用的功能 |

## 演示后 Q&A 预期问题

| 问题 | 回答要点 |
|------|----------|
| 数据安全吗？ | 全部本地存储（chrome.storage.local + SQLite），API 调用只发送必要文本 |
| 支持其他语言吗？ | 界面中英文，翻译支持多语言，核心功能语言无关 |
| 什么时候上架？ | 功能完善中，计划提交 Chrome Web Store |
| 能处理中文论文吗？ | 完全可以，PDF.js 支持中英文混排 |
| 和 ChatGPT / Claude 有什么不同？ | Impulse 是**可操作的**——不只聊天，能操控页面（高亮、笔记、对比）|
| 需要自己部署吗？ | 不需要，就是浏览器扩展，配置 API Key 即可 |

## 关键信息备忘

- **项目名称**：Impulse
- **Slogan**：Read less, understand more. 读得更少，理解更多。
- **技术栈**：Plasmo + React 18 + TypeScript 5 + pdfjs-dist + sql.js
- **核心差异点**：Agent-first（不只是工具）、Goal-aware（阅读目标驱动）、Action-oriented（能操控页面）
- **工具数量**：14 个 Agent 工具 + 9 个技能
- **高亮分类**：5 种颜色（重要🔴 疑问🔵 定义🟢 方法🟣 默认🟡）
